git常用命令

git会随时对比三棵树（工作目录，暂存区，仓库快照）状态的一致性
创建一个项目文件夹，git init 初始化
在工作区添加文件（README.md）

git add 文件名 添加到暂存区

git commit -m “描述” 提交到仓库

git status 查看状态

红色，Changes not staged for commit:未添加到暂存区，绿色，Changes to be committed:未提交到仓库

git reset HEAD 将仓库的状态回退到暂存区的状态

git checkout – 文件名 将暂存区的状态覆盖工作目录的状态

git checkout 文件名 将暂存区的文件覆盖到工作目录（同上，确保没有一个和该文件同名的分支）

git checkout HEAD~ 文件名 从上一个快照中拷贝文件到当前工作目录和暂存区

git log 查看提交日志

git reset HEAD~ 移动HEAD的指向（~表示HEAD指向的快照的上一个版本快照）回滚到该快照的暂存区（在该暂存区可能检测到当前工作目录的新文件（回滚之前的））

git reset [–mixed] HEAD~ 移动HEAD的指向，指向上一个快照，将该快照回滚到暂存区（默认可不写mixed）

git reset –soft HEAD~ 移动HEAD的指向，指向上一个快照（相当于撤销错误提交）

git reset –hard HEAD~ 移动HEAD的指向，指向上一个快照，将该快照回滚到暂存区，并将暂存区的文件还原到工作目录（覆盖）

git reset 版本快照id（前5个字母就可以） 将快照滚到某次

git reset 版本快照id 文件名/路径(忽略HEAD指针，指针不会改变)

git diff 比较暂存区和工作目录的内容 -表示历史（暂存区），+表示新的（工作目录）

git commit -am “描述” 一次性添加到暂存区并提交到仓库

git diff 快照id1 快照id2 对比两次快照的内容

git diff 快照id 比较指定快照和当前工作目录的内容

git diff HEAD 比较最新版本快照和当前工作目录的内容

git diff –cached 比较仓库最新快照和暂存区的内容

git diff –cached 快照id 比较指定仓库快照和暂存区的内容

git commit –amend 修改提交说明（vim操作)（不会产生新的快照）

git commit –amend -m “提交说明” 中文说明修改

git rm 文件名 删除工作目录和暂存区的文件，在下次提交是不会纳入版本管理

git rm -f 文件名 强制删除工作目录和暂存区的文件（可能暂存区和工作目录的该文件内容不一样）

git rm –cached 文件名 删除暂存区的文件

git mv 旧文件名 新文件名 重命名文件（不会改变快照）

git baranch 分支名称 创建分支

git log –decorate 查看HEAD所指向的那个分支的快照(HEAD指针默认指向master）

git log –decorate –oneline 查看HEAD所指向的那个分支的快照（简化快照信息显示）

git log –decorate –oneline –graph –all 查看HEAD所指向的那个分支的快照（简化快照信息显示） 图形化显示所有分支指向的快照体系

git checkout 分支名 切换分支（HEAD指向该分支）HEAD指向的分支的快照中的文件会拷贝到工作目录和暂存区。目录和暂存区中切换前的内容会被删掉。

git checkout -b 分支名 创建新分支并切换到该分支

git merge 分支名 合并分支到 当前分支（HEAD所指向的分支）

git branch -d 分支名 删除分支（不会删除快照，因为分支是通过指针来快速完成的）

git checkout HEAD~ 切换到上一个快照并在此开起匿名分支，在该匿名分支下可以做各种测试操作并提交，一旦切换回有名字的分支，该匿名分支所做的操作提交不会有任何效果。

git reset –hard 分支名 将HEAD指向的分支以及HEAD本身切到了该分支里，那么原来的快照就会消失了
