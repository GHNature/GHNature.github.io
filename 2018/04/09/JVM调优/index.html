jvm调优

jvm生命周期：

以下几种情况将导致jvm进程结束，

  1. 在程序中调用System.exit();

  2. 在程序运行过程中出现exception或error

  3. 操作系统导致jvm进程结束

  4. 程序正常执行完毕

jvm对类的装载过程：

  1. 加载
    查找并加载类的二进制数据，把.class文件加载到内存中，将其放在运行时数据区的方法区中，然后在堆中创建一个java.lang.Class对象，用来封装类在方法区中的数据结构。

  2. 连接
    验证：确保被加载的class文件的正确性

准备：为类的静态变量分配内存，并初始化为默认值


解析：将类中的符号引用转为直接引用


  1. 初始化
    给类的静态变量赋予正确的(显示)初始化值


  • 类的初始化步骤：

java程序对类的使用方式有两种:

  1. 主动使用

  2. 被动使用

  • 所有的jvm实现必须在每个类或接口被java程序首次主动使用时才初始化它们。除了主动使用，其他使用类的方式被视为被动使用，不会导致类的初始化。

  • 主动使用分为以下六种情况:

  1. 创建类的实例(new)

  2. 访问某个类或接口的静态变量或对该变量赋值

  3. 调用类的静态方法

  4. 反射(Class.forName(“”))

  5. 初始化一个类的子类，该类也会被初始化

  6. jvm启动时被标明为启动类的类(如包含main方法的Test类，java Test)

加载.class文件的方式:

  1. 从本地系统直接加载

  2. 通过网络下载.class文件(如URLClassLoader)

  3. 从zip，jar等归档文件中加载.class文件

  4. 从专有数据库中提取.class文件

  5. 将Java源文件动态编译为.class文件

  • 只有当程序访问的静态变量或静态方法确
    实在当前类或当前接口中定义时，才可以
    认为是对类或接口的主动使用。

  • 调用ClassLoader类的loadClass方法加载
    一个类，并不是对类的主动使用，不会导
    致类的初始化。

类加载器：

  • 1. jvm自带的类加载器

  • 根类加载器(Bootstrap)

由c++编写，用来加载核心包下的类，每个类的Class对象都包含一个对加载本类的类加载器的引用，可通过代表该类的Class对象的getClassLoader()获得加载自己的类加载器。如果本类是由Bootstrap加载，将返回null。

  • 扩展类加载器(Extension)

  • 系统类加载器(System)

  • 2. 用户自定义的类加载器

java.lang.ClassLoader的子类，用户可以自定义类的加载方式。

  • JVM规范允许类加载器在预料某个类将要被使用时就预先加载它，如果在预先加载过程中遇到.class文件缺失或存在错误，类加载器必须在程序首次主动使用该类时，才报告错误(LinkageError，比如在jdk1.6编译的.class放到jdk1.5中运行，也会报这个错误)，如果这个类一直没有被程序主动使用，类加载器就不会报告错误。

创建用户自定义的类加载器：


新建一个java项目，在默认包下创建类MyclassLoader,Sample,Dog。
在windows硬盘上创建d:\myapp\serverlib，d:\myapp\clientlib，
d:\myapp\otherlib，d:\myapp\syslib，将编译好的.class根据测试需要放到相应目录下

MyClassLoader.java

import java.io.ByteArrayOutputStream;
import java.io.File;
import java.io.FileInputStream;
import java.io.InputStream;
import java.lang.reflect.Field;

/**
 * 自定义类加载器
 * @author Nature
 *
 */
public class MyClassLoader extends ClassLoader
{
    static int i = 0;    // 计数该类被加载的次数
    static{
        System.out.println("MyClassLoader被加载第"+ ++i +"次");//注意：一个类只会被同一个类加载器加载一次。若被不同的类加载器加载，会存在各自的命名空间中。

        String s = "String is loaded by ";    // String类由BootStrap加载器加载，将返回null
        System.out.println(s + s.getClass().getClassLoader());
    }

    private String name;    // 类加载器的名字
    private String path = "d:\\";    // 要加载的类的路径
    private final String fileType = ".class";    // 扩展名为class

    public MyClassLoader(String name){
        super();            // 让系统类加载器成为该类加载器的父加载器
        this.name = name;
    }

    public MyClassLoader(ClassLoader parent, String name){
        super(parent);        // 指定该类加载器的父加载器
        this.name = name;
    }

    // ClassLoader中此方法默认实现是抛出异常，覆盖此方法，当调用ClassLoader的loadClass方法，会自动调用此方法
    @Override
    public Class<?> findClass(String name) throws ClassNotFoundException
    {
        byte[] data = this.loadClassData(name);
        return defineClass(name,data,0,data.length);
    }

    // 根据类全名"com.nature.className"到硬盘上找到相关的.class文件，将其转为字节数组
    private byte[] loadClassData(String name){
        InputStream is = null;
        byte[] data = null;
        ByteArrayOutputStream baos = null;
        try{
//            name = name.replace(".", "\\");
            is = new FileInputStream(new File(path + name + fileType));
            baos = new ByteArrayOutputStream();
            int ch = 0;
            while((ch=is.read())!=-1){
                baos.write(ch);
            }
            data = baos.toByteArray();

        }catch(Exception e){
            e.printStackTrace();
        }finally{
            try{
                is.close();
                baos.close();
            }catch(Exception e){
                e.printStackTrace();
            }

        }
        return data;
    }

    @Override
    public String toString()
    {
        return this.name;
    }

    public void setPath(String path){
        this.path = path;
    }

    public String getPath(){
        return this.path;
    }

    public static void main(String[] args) throws Exception {    // 作为启动类
        MyClassLoader loader1 = new MyClassLoader("loader1");// loader1的父加载器为系统（应用）类加载器
        loader1.setPath("d:\\myapp\\serverlib\\");
        MyClassLoader loader2 = new MyClassLoader(loader1, "loader2");// loader2的父加载器为loader1
        loader2.setPath("d:\\myapp\\clientlib\\");
        /*
        MyClassLoader loader3 = new MyClassLoader(null, "loader3");    // loader3的父加载器为Bootstrap
        loader3.setPath("d:\\myapp\\otherlib\\");

        test(loader2);
        System.out.println("----------------");
        test(loader3);

        // Sample由loader1加载
        Class clazz = loader1.loadClass("Sample");
        Object object = clazz.newInstance();
        // MyClassLoader由系统类加载器加载，Sample和此类属于不同的命名空间，不能互相访问数据
        Sample sample = (Sample) object;// java.lang.NoClassDefFoundError
//        System.out.println(sample.v1);    // java.lang.NoClassDefFoundError
//        new Sample();                    // java.lang.NoClassDefFoundError
        */

        /*
        // 通过反射解决不同命名空间（非父子加载器）下的属性访问。
        Class clazz = loader1.loadClass("Sample");
        Object object = clazz.newInstance();
        Field field = clazz.getDeclaredField("v1");
        field.setAccessible(true);
        int v1 = field.getInt(object);
        System.out.println("v1:" + v1);
        */

        Class clazz = loader1.loadClass("Sample");
        System.out.println(clazz.hashCode());    // 1550089733
        Object object = clazz.newInstance();
        loader1 = null;
        clazz = null;
        object = null;
        loader1 = new MyClassLoader("loader1");
        loader1.setPath("d:\\myapp\\serverlib\\");
        clazz = loader1.loadClass("Sample");
        System.out.println(clazz.hashCode());    // 1028566121
    }

    public static void test(ClassLoader loader) throws Exception {
        Class clazz = loader.loadClass("Sample");
        Object object = clazz.newInstance();
    }

}

}
