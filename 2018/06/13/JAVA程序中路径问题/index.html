java程序中路径问题

java程序中的路径书写

在书写路径时，首先分析是服务器行为还是浏览器行为。
服务器行为： / 代表项目根目录
浏览器行为： / 代表站点根目录

.代表当前路径

java命令在哪个目录下运行，那个目录就代表当前目录。

在MyEclipse中新建一个java项目Test_Path，新建一个java文件Test_Path，代码如下： 

package gz.itcast;

import java.io.File;

public class TestPath {
    public static void main(String[] args) {
        // 查看当前目录所在目录
        File file = new File(".");
        System.out.println(file.getAbsolutePath());
    }
}

运行结果为： 

C:\Users\Nature\Workspaces\MyEclipse 8.5\Test_Path\.

打开cmd窗口进入以上路径，运行java命令： 

cd C:\Users\Nature\Workspaces\MyEclipse 8.5\Test_Path\
java gz.itcast.TestPath

会出现： 

错误: 找不到或无法加载主类 gz.itcast.TestPath

我们再进入下一级目录： 

cd bin
java gz.itcast.TestPath

结果为： 

C:\Users\Nature\Workspaces\MyEclipse 8.5\Test_Path\bin\.
  • 为什么会这样呢？

在MyEclipse中，对于java工程，所有编译好的class文件产生在项目根目录的bin目录中，
如果直接在MyEclipse中运行，该工具会自动把java命令指向到项目的根目录。也就是说在项目根目录下运行的java命令，而执行的类路径的class文件。
如果在cmd中运行，需要进入该bin目录运行java命令，或者手动指定classpath。
（注意：java命令运行时一定要书写类的全名）
对于web工程，java命令运行的目录在tomcat的bin目录，所以当前路径.就是该bin目录。

在web项目中，所有的编译好的class文件生成在tomcat服务器应用目录下的WEB-INF/classes目录下
所以获取文件的时候，不要使用相对路径（相对于当前目录.）。

通过ServletContext获取文件的路径

ServletContext的getRealPath() 方法获取文件的真实路径。如： 

String path = this.getServletContext().getRealPath("/WEB-INF/classes/news.properties");

这样就可以获取到文件的绝对路径。

ServletContext的getResourceAsStream() 获取资源文件的输入流，如： 

InputStream in = this.getServletContext().getResourceAsStream("/WEB-INF/classes/news.properties");

ServletContext的getResource() 获取资源文件的URL，如： 

URL url = this.getServletContext().getResource("/WEB-INF/classes/news.properties");
String path = url.getPath();

获取资源文件的URL地址。/localhost/应用名/WEB-INF/classes/news.properties
