Android运行Ubuntu

Android运行Ubuntu

  • 本文章主要是介绍搭建步骤

所需环境与工具：

Android，LinuxDeploy，Xserver XSDL，JuiceSSH

拥有安卓的你，有没有想过在此设备上折腾一下linux系统呢，当然，其本身也是基于linux的，不过，为了获得更强大的原生linux操作，还需要进行以下的工作。
方法不是唯一的，大致原理是不变的。现在通过linuxdeploy可以获得linux不同架构的各个发行版，以上应用均可在GooglePlay Store获得最新版。

步骤一:

首先，打开linuxdeploy，要安装什么系统自己根据自己情况，需求选择。这里以Ubuntu 18.04作记录。
软件设置，参考下图。源地址选择清华的源，http://mirrors.tuna.tsinghua.edu.cn/ubuntu-ports，挂载点如：/sdcard:/mnt/sdcard

按照上图，安装完成后，你的系统在你设备中的真实路径为/data/data/ru.meefik.linuxdeploy/env/rootfs/linux/。
安装完系统后，用户名，密码若再次更改需要点击配置一下。

步骤二:

上述安装无误后，在进入桌面环境前，可以先为系统进行一些安装，启动系统，打开SSH，配置连接，IP为127.0.0.1，认证所需的用户名和密码来自linuxdeploy里面设置的。
成功连接到系统后，可以通过，uname -a查看系统版本。开始安装一些软件。 

sudo su // 获取root权限
sudo apt-get remove --purge mate* // 删除MATE桌面环境
sudo apt-get autoremove // 删除相关软件
apt-get update // 更新软件源
apt-get upgrade // 更新软件
apt-get install ubuntu-mate-core // 安装MATE桌面核心
apt-get install ubuntu-mate-desktop // 安装MATE桌面环境
sudo apt-get install ubuntu-desktop // 安装ubuntu桌面一些东西

桌面安装完只后再安装中文 

sudo apt-get install xfonts-intl-chinese
sudo apt-get install ttf-wqy-microhei

步骤三:

完成后，linuxdeploy关掉系统，打开XSDL，直接点击确定，全部默认设置，出现黑窗口时，点击，选择分辨率，字体大小。推荐，19201080/0.4~0.5或者1024768/0.5自己尝试。每次软件启动都可以设置。
出现蓝屏后，后台，回到linuxdeploy，启动系统，启动无误后，回到XSDL，此时就顺利进入桌面环境了。

  • 补充：没有声音的，在ssh中完成以下操作。 
    sudo apt-get install pulseaudio pavucontrol
sudo apt-get install vim
sudo vim /etc/profile // 在末尾新起一行添加export PULSE_SERVER=tcp:localhost:4712，保存退出。
  • VLC播放器视频花的话，可以用mpv 
    sudo apt-get install mpv
    至此，ubuntu安装结束，慢慢摸索。

Ubuntu安装Oracle JDK

一、获取jdk

下载好arm架构的jdk，可在Oracle官网获取jdk-8u181-linux-arm64-vfp-hflt.tar.gz

二、解压jdk压缩包

新建目录: 

sudo mkdir /usr/lib/jvm

目录创建成功后解压缩到该目录: 

sudo tar -zxvf jdk-8u181-linux-arm64-vfp-hflt.tar.gz -C /usr/lib/jvm

三、配置jdk环境产量，classpath

vim ~/.bashrc

在文件末尾新起一行，配置以下内容: 

# set oracle jdk environment 这里JAVA_HOME对应解压的jdk 目录
export JAVA_HOME=/usr/lib/jvm/jdk1.8.0_181
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib
export PATH=${JAVA_HOME}/bin:${PATH}

保存退出，使文件修改生效:source ~/.bashrc

查看java版本:java -version
至此，java安装成功。

设置默认jdk

有时系统中有多个jdk(一些软件安装时默认安装的)，除了设置环境变量的方式，也通过Ubuntu下update-alternatives命令(介绍:维护系统命令链接符的工具，其可以对某个工具的多个软件版本进行管理，通过它可以很方便的设置系统默认使用哪个命令的哪个软件版本。原理:/usr/bin/<name>这个软链接，指向了/etc/alternatives/<name>这个文件，其也是个软链接，指向了该<name>命令的实际可执行文件)添加软链接以添加java可选配置项，完成jdk版本切换。

添加软链接:

sudo update-alternatives --install /usr/bin/java java /usr/lib/jvm/jdk1.8.0_181/bin/java 300
sudo update-alternatives --install /usr/bin/javac javac /usr/lib/jvm/jdk1.8.0_181/bin/javac 300
sudo update-alternatives --install /usr/bin/jar jar /usr/lib/jvm/jdk1.8.0_181/bin/jar 300
sudo update-alternatives --install /usr/bin/javah javah /usr/lib/jvm/jdk1.8.0_181/bin/javah 300
sudo update-alternatives --install /usr/bin/javap javap /usr/lib/jvm/jdk1.8.0_181/bin/javap 300

然后切换: 

sudo update-alternatives --config java
sudo update-alternatives --config javac

输入对应选项的编号即可，若不改变可直接回车
再输入java -version查看。
