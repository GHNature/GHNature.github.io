使用Gradle构建Spring boot项目

使用Gradle构建Spring boot项目

第一个springboot项目Spring INITIALIZR

登录https://start.spring.io初始化springboot项目。

下载gradle，进入https://gradle.org/，下载后配置好gradle环境变量。

打开cmd，java -version，gradle -v，进入initializr-start目录，开始构建gradle build，构建成功后。项目根目录多了build/libs目录，有了一个构建生成的jar文件。回到cmd，执行java -jar build/libs/initializr-start-0.0.1-SNAPSHOT.jar，启动成功后，访问localhost:8080得到如下错误页面，因为还未在项目中写自己的代码。

编写Hello World

新建文件夹hello-world，拷贝相关文件到该目录下。

可以把版本号改短一点，编辑build.gradle，更改version = '1.0.0'保存。
进入cmd该路径下，进行编译。执行gradle build，然后运行该项目java -jar build/libs/initializr-start-1.0.0.jar，运行成功。

配置阿里云Maven，打开项目下的build.gradle文件，将maven中心仓库注释，添加阿里云maven { url 'http://maven.aliyun.com/nexus/content/groups/public/' }，保存。

通过gradle wrapper编译运行。将项目下gradle/wrapper/gradle-wrapper.properties打开编辑，distributionUrl指定gradle版本。打开cmd，进入项目根目录，执行gradlew build编译，然后java -jar 运行。

在IDE中配置Gradle插件并导入项目运行

这里使用的是eclipse4.x，jdk1.8，gradle4.7。
打开IDE，配置jdk安装路径。

点击help->Install New Software…，下载安装Gradle插件。




也可以通过Eclipse MarketPlace搜索Gradle安装。

Gradle插件安装完成后，eclipse提示重启使插件生效，重启后打开Window->Preferences。可以看到已经有Gradle了，配置Gradle User Home为之前解压的Gradle的根目录。

至此Gradle在Eclipse中配置完成。

在Eclipse中导入Gradle项目，选择gradle构建过的项目，


Gradle distribution选择Gradle wrapper以统一项目开发环境，避免不必要的错误，然后next。

完成后，选中项目，点击finish，项目导入完成。

SpringBoot项目的三种运行方式：

  1. java -jar 命令
  2. 以”Java Application”运行
  3. Spring Boot Gradle Plugin 插件

第三种方式为：打开cmd，进入项目根目录，执行gradle bootRun或者以gradle wrapper运行gradlew bootRun。
